Inputs and Requirements

The automation expects three business extracts. Each file is validated before loading so missing columns and blank required values are caught early.

Sales Orders

data/raw/sales_orders.csv

  • One row per order line
  • Uses completed sales only
  • Fields: order ID, date, SKU, quantity, unit price, channel, status

Inventory Snapshot

data/raw/inventory_snapshot.csv

  • One row per SKU
  • Contains opening stock, receipts, system close, and warehouse count
  • Used as the main stock control file

Warehouse Adjustments

data/raw/warehouse_adjustments.csv

  • One row per adjustment
  • Separates posted and unposted changes
  • Captures the adjustment reason for review

Software Requirements

  • Python 3.10 or newer
  • SQLite through Python standard library
  • No external packages required

Output Files

Business Rules Implemented

Rule Automation Behavior
Cancelled sales excluded Only completed orders are included in sold quantity and sales value.
Missing inventory record SKU appears in sales but not in the inventory snapshot.
Oversell risk Sold quantity is greater than available inventory.
Unposted adjustment Warehouse adjustment exists but has not been posted to ERP.
System stock mismatch System closing stock differs from calculated expected stock.
